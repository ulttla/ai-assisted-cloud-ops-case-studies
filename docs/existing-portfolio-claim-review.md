# Existing Portfolio Claim Review

This note is for final review before publishing new AI-assisted case-study content alongside the existing gunkr.com portfolio.

## Why this exists

The new AI-assisted content is sanitized. However, it appears next to existing portfolio claims about employer projects, security incidents, uptime, budgets, migrations, and cost savings. Those claims may be acceptable if already approved for public portfolio use, but they should be rechecked before a job-search push.

## Claims to re-confirm

- Employer name and current role wording
- Budget ownership numbers
- SLA or uptime claims
- Security incident / ransomware recovery wording
- Zero data loss or zero incident claims
- Mailbox and data migration quantities
- Cost savings percentages or dollar values
- Any customer, location, or acquisition details

## Safer rewrite pattern

| Higher-risk wording | Safer public wording |
|---|---|
| Exact budget ownership | Managed infrastructure planning and vendor coordination across a lean IT environment |
| Ransomware incident specifics | Led recovery and resilience improvements after a major security incident |
| Zero data loss / zero incidents | Completed migrations without material business disruption |
| Exact mailbox/TB counts | Migrated large-scale mail and file data across multiple offices |
| Exact savings | Reduced infrastructure and operations costs through cloud and platform modernization |

## Recommendation

Before pushing a major portfolio refresh, decide whether existing claims should stay exact, be footnoted with evidence, or be softened into public-safe wording.
